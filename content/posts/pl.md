A beginners main concern is to develop problem solving skills that allow them to design high quality programs. While one can learn to program with an industry level language like Python or C++, it should be noted that these languages were not designed with the beginner in mind and hence make the process of developing problem solving skills more difficult.

>Learning to design programs is primarily about the study of principles and the acquisition of transferable skills. The ideal programming language must support these two goals, but no off-the-shelf industrial language does so. The crucial problem is that beginners make mistakes before they know much of the language, yet programming languages always diagnose these errors as if the programmer already knew the whole language. How to Design Program 2nd edition

As the quote suggests, what is really needed is a programming language specifically designed to teach people how to program. Thanks to the efforts of some amazing people, such languages exist. The ones that I have years of experience writing code in and teaching include Scratch, [Pyret](https://www.pyret.org/) and [BSL](https://htdp.org/2020-8-1/Book/index.html)(beginner student language).

These languages allow students to focus on problem solving which lies at the core of writing computer code rather than the nuances and quirks of a particular programing language.

For more discussion on this topic please see [how to design programs.](https://htdp.org/2020-8-1/Book/part_preface.html#%28part._drtl%29)
